# Data Engineering Practices

Репозиторий содержит набор проектов, демонстрирующих ключевые паттерны и подходы в инженерии данных. Проекты охватывают построение надежных REST API, интеграцию с брокерами сообщений, кэширование запросов, потоковую обработку событий, разработку ETL-пайплайнов и настройку мониторинга.

## Технологический стек
* **Язык программирования:** Python 3.10+
* **Web-фреймворк:** FastAPI, Uvicorn, Pydantic
* **Обработка данных (ETL):** Pandas
* **Брокеры сообщений и стриминг:** RabbitMQ, Apache Kafka
* **Базы данных и кэширование:** SQLite, Redis
* **Мониторинг:** Prometheus
* **Инфраструктура:** Docker, Docker Compose

## Структура проекта

Каждая директория в репозитории представляет собой независимый микро-проект, решающий конкретную инфраструктурную задачу:

* **[01_rabbitmq](./01_rabbitmq)** 
  Реализация асинхронной обработки сообщений. Содержит FastAPI-приложение (Producer), отправляющее задачи, и отдельный воркер (Consumer), который читает их из очереди RabbitMQ.

* **[02_redis](./02_redis)** 
  Оптимизация производительности с помощью кэширования. Демонстрирует паттерн Cache-Aside: FastAPI сначала проверяет наличие данных в Redis, и только при промахе (cache miss) обращается к основной базе данных. Включает скрипты для нагрузочного тестирования.

* **[03_kafka](./03_kafka)** 
  Потоковая передача событий (Event Streaming). Настроен `docker-compose` с Zookeeper и Kafka. Реализованы сервисы для записи логов/событий через FastAPI в топик Kafka и их асинхронного вычитывания консьюмером.

* **[04_prometheus](./04_prometheus)** 
  Настройка наблюдаемости (Observability). Реализован кастомный Middleware для FastAPI, который собирает бизнес-метрики (RPS, Latency) и отдает их в формате, понятном для Prometheus.

* **[05_etl_sqlite](./05_etl_sqlite)** 
  Разработка ETL-пайплайна (Extract, Transform, Load). Эндпоинт принимает сырой CSV-файл, с помощью Pandas производит очистку данных, валидацию типов, фильтрацию по бизнес-правилам и загрузку чистых данных в реляционную БД (SQLite).

## Как запустить проекты

Для запуска инфраструктуры (Redis, RabbitMQ, Kafka, Prometheus) используется Docker. Базовый алгоритм запуска выглядит так:

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/SabiaPI1/data-engineering-practices.git
   cd data-engineering-practices
   ```
2. Поднять необходимые контейнеры (на примере Kafka):
   ```bash
   cd 03_kafka
   docker-compose up -d
   ```
3. Установить зависимости и запустить приложение:
   ```bash
   pip install -r requirements.txt
   uvicorn app.main:app --reload
   ```
